# mrs-buttons-kitchen
AI-powered recipe search web application

AI와 YouTube를 결합한 레시피 검색 웹 애플리케이션

## 개발 배경 및 동기

요리를 만들고 싶을 때 인터넷에서 레시피를 검색하면 너무 많은 결과가 한꺼번에 쏟아져 원하는 레시피를 찾는 데 오랜 시간이 걸린다. 또한 다수의 레시피가 그램(g)·밀리리터(mL) 단위로 재료를 계량하기 때문에 계량 도구 없이는 실제 조리에 적용하기 어렵다는 불편함이 있다.

이러한 문제를 해결하기 위해 키워드 검색 한 번으로 YouTube 레시피 영상과 AI가 생성한 레시피를 한 화면에서 비교·확인할 수 있고, 재료 계량은 큰술·작은술·컵·개 등 가정에서 바로 쓸 수 있는 단위로만 제공하는 웹 애플리케이션을 개발하였다.

---

## 웹사이트 주제

Mrs. Button's Kitchen은 AI와 YouTube를 결합한 레시피 검색 서비스다. 사용자가 요리 이름을 검색하면 관련 YouTube 영상과 Gemini AI가 생성한 다양한 스타일의 레시피(지역별·건강식·퓨전 등)를 동시에 제공하며, 마음에 드는 레시피는 저장해 언제든 다시 확인할 수 있다.

## 기술 스택

- Frontend: HTML5, CSS3, Vanilla JavaScript (SPA 구조)
- AI: Google Gemini API (gemini-3.1-flash-lite)
- 영상: YouTube Data API v3

## 구현 방법

### SPA 구조

페이지 이동 없이 하나의 HTML 파일 안에서 탭을 전환하는 SPA 방식을 적용하였다. index.html에는 홈·탐색·추천·저장됨·프로필·로그인·회원가입 화면이 각각 `.page` 클래스를 가진 div 블록으로 존재하며 기본적으로 모두 숨겨진 상태다. `goPage(id)` 함수가 호출되면 현재 활성 페이지에서 `.active` 클래스를 제거하고 대상 페이지에만 추가하는 방식으로 화면을 전환한다.

### Gemini API 연동

`callGemini()` 함수는 API 요청 초과(HTTP 429)가 발생하면 지수 백오프 방식으로 재시도한다. 1회 실패 시 1초, 2회 실패 시 2초를 기다린 뒤 재호출하며, 3회 모두 실패하면 에러를 던지고 검색 버튼에 60초 카운트다운을 표시한다. Gemini 응답은 JSON 외에 부가 텍스트가 섞여 올 수 있어 정규식으로 JSON 객체 부분만 추출한 뒤 파싱한다.

### YouTube 검색 — 중복 제거 및 채널 편향 방지

한국어(`{검색어} 레시피 만드는법`)와 영어(`{검색어} recipe cooking`) 두 쿼리로 동시에 검색한 뒤 `Promise.all()`로 병렬 처리한다. 두 목록을 합친 후 Set을 이용해 videoId 기준 중복을 제거하고, 결과가 4개 이상일 경우 채널당 최대 2개만 남겨 특정 채널이 목록을 독점하는 현상을 방지한다.

### 계량 단위 통일

Gemini에게 레시피 생성을 요청할 때 프롬프트에 "재료 계량은 큰술/작은술/컵/개 단위만 사용, g/mL 금지"를 명시적으로 지시한다. 스타일 4가지를 한 번의 API 호출로 받아오도록 하여 불필요한 호출 횟수를 줄였다.

### 오늘 메뉴 추천

12가지 음식 종류와 8가지 분위기 배열에서 랜덤으로 각각 하나씩 뽑아 Gemini에게 추천을 요청한다. 매 요청마다 무작위 seed 값을 함께 전달해 같은 음식이 반복 추천되는 현상을 방지하며, 추천 결과를 화면에 표시한 뒤 해당 음식 이름으로 레시피 카드까지 자동으로 이어 검색한다.

### 저장 기능

백엔드 서버 없이 브라우저의 localStorage를 활용해 데이터를 영구 보존한다. YouTube 영상은 `savedYT`, AI 레시피는 `savedAI` 키에 JSON 배열로 저장되며, AI 레시피는 고유 ID 대신 title을 기준으로 중복을 판별한다. 메모는 `yt_{videoId}` 또는 `ai_{title}` 형태의 키로 `recipeMemos` 객체에 저장되어 여러 레시피의 메모가 하나의 항목으로 관리된다.

### 회원 기능

회원 정보도 localStorage에 저장한다. 회원가입 시 아이디·비밀번호·이름·이메일을 정규식으로 검증하며 모든 항목이 통과해야 가입 처리된다. 프로필 사진은 FileReader API로 base64 문자열로 변환해 저장하고 네비게이션 바 아바타에 즉시 반영된다.

## 한계 및 고려 사항

- 클라이언트 사이드 인증: 로그인 정보가 localStorage에만 저장되어 실제 보안은 제공되지 않는다. 실제 서비스 전환 시 서버 세션 또는 JWT 기반 인증이 필요하다.
- API 키 노출: Gemini·YouTube API 키가 프론트엔드 JS 파일에 포함되어 있어 운영 환경에서는 백엔드 프록시를 통해 키를 숨겨야 한다.
- Rate Limit: Gemini 무료 티어의 분당 요청 제한으로 빠른 연속 검색 시 대기 시간이 발생할 수 있다.
- localStorage 용량: 프로필 사진을 base64로 변환해 저장하므로 고해상도 이미지 등록 시 용량(일반적으로 5MB)이 빠르게 소진될 수 있다.
